intuition

To build a type theoretic narrative, we start with the natural types from implementation architectures. As any given architecture differs, we focus on cross-platform specifications such as register (or sub-register) lengths to generate our infrastructural types. To put this another way, we look to specify the common typed patterns seen across assembly languages. The idea of a register type is one such pattern.

In this library, register types (RegType) are intended to correspond to any of the following C++ builtin types:

(un)signed char
(un)signed short
(un)signed int
(un)signed long
(un)signed long long
